# Phase 1 — UI Design Contract

> Visual and interaction contract for Phase 1: KPI Dashboard + PDF Export.
> Generated by gsd-ui-researcher. Consumed by gsd-planner, gsd-executor, gsd-ui-checker, gsd-ui-auditor.

**Platform:** Flutter 3.41.6 / Dart 3.11.4 — Android, Web, Windows. Dark theme only. Material Design 3 via `IndustrialTheme.dark`.

**Constraint:** Extend existing screens. Do NOT restyle or restructure any screen section not listed in this contract.

## Spacing Scale

All Flutter padding/SizedBox values must use this scale. Source: measured from existing `kpis_screen.dart` and `dashboard_screen.dart`.

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4 | Icon gaps, badge padding, inline separators |
| sm | 8 | Gap between icon and text, compact row spacing |
| md | 16 | Default card internal padding (`padding: const EdgeInsets.all(16)`) |
| lg | 24 | Card internal padding for ratio/estado containers, section header spacing |
| xl | 32 | Between KPI sections (`SizedBox(height: 32)`) |
| 2xl | 40 | Bottom of scroll content (`SizedBox(height: 40)`) — codebase-measured exception |

Exceptions:
- Card `BorderRadius.circular(16)` — all cards, everywhere, no exceptions.
- `BorderRadius.circular(12)` — buttons and inputs only.
- `BorderRadius.circular(8)` — small badge containers.
- `BorderRadius.circular(4)` — progress bar clips only.
- Touch targets for icon buttons: minimum 44×44 logical pixels (Flutter default `IconButton` satisfies this automatically).

## Typography

Source: `industrial_theme.dart` — Montserrat from google_fonts. All text is ALL CAPS for labels and headings per existing app convention.

**Flutter widget typography — exactly 4 sizes, exactly 2 weights:**

| Role | Size | Weight | Line Height | Usage |
|------|------|--------|-------------|-------|
| Display value | 22px | bold (w700) | 1.0 | KPI card primary value (e.g. "73.2%", "43.8h"), sensor card value |
| Heading / AppBar / section header | 14px | bold (w700) | 1.0 | AppBar title (`letterSpacing: 2`), screen section titles (`letterSpacing: 1.1`) |
| Label / card title | 10px | bold (w700) | 1.0 | KPI card label line ("OEE", "MTBF", "SALA DE SERVIDORES") — `letterSpacing: 1` |
| Body / subtitle / legend | 9px | regular (w400) | 1.0 | KPI card subtitle line ("Eficiencia Global Equipos"), secondary data, legend labels, live badge ("EN VIVO"), axis labels |

> **PDF-only exception (excluded from Flutter widget table):** The `pdf` package (pw.*) document headers use `fontSize: 18` and `fontSize: 16` for report titles. These values exist solely in `PdfGenerator` methods and are never rendered as Flutter widgets. They do not count toward the 4-size Flutter constraint.

Color rule for all text roles:
- Primary text: `Colors.white`
- Secondary text: `IndustrialTheme.slateGray` (#8892B0)
- Accent text (live data, active states): `IndustrialTheme.neonCyan` (#00E5FF)
- State-colored text: use the semantic color of that state (operativeGreen, warningOrange, criticalRed)

## Color

Source: `industrial_theme.dart` — verified from codebase.

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | `#0A192F` (`spaceCadet`) | `scaffoldBackgroundColor`, screen backgrounds, AppBar background |
| Secondary (30%) | `#112240` (`claudCloud`) | All cards (`cardColor`), Drawer background (`spaceCadet`), container backgrounds |
| Accent (10%) | `#00E5FF` (`neonCyan` / `electricBlue`) | See reserved list below |
| Semantic — operative | `#00C853` (`operativeGreen`) | Preventive OT bars, healthy KPI indicators, CERRADA OT state bar, live indicator dot when data is fresh |
| Semantic — warning | `#FFA500` (`warningOrange`) | PENDIENTE OT state bar, warning KPI thresholds, ratio objective not-met banner |
| Semantic — critical | `#D32F2F` (`criticalRed`) | Corrective OT bars, alert cards, destructive drawer item |

Accent (`neonCyan`) is reserved strictly for:
- AppBar icon color (`iconTheme`)
- Drawer active item icon
- `CircularProgressIndicator` color
- `RefreshIndicator` color
- Live/online status dot on `SalaServidoresWidget`
- `Icons.refresh` buttons in AppBar actions
- Role badge text in Drawer header
- `Icons.picture_as_pdf` export button in OrdenesScreen AppBar
- `focusedBorder` on inputs

Accent is NOT used for: generic body text, card backgrounds, chart bars, heading text, or section titles.

Bar chart semantic assignment:
- Preventive OTs bars: `operativeGreen` (#00C853)
- Corrective OTs bars: `criticalRed` (#D32F2F)

These are locked. Do not invert or reuse for other chart series.

## Component Inventory — Phase 1 Surfaces

### Surface 1: KPI Mini-Cards Row (KPI-01) — DashboardScreen

**What:** New row of 4 stats-based KPI cards added to DashboardScreen body, inserted between the existing 2×2 operational card grid and the INCIDENCIAS ACTIVAS section.

**Widget pattern:** Reuse `_buildKpiCard(title, value, icon, color)` already present in `DashboardScreen`. Build two `Row` widgets, each containing two `Expanded` cards with `SizedBox(width: 12)` between them. Follow the exact same pattern as the existing operational card rows.

**Card dimensions:**
- Padding: `EdgeInsets.all(16)` (md token)
- BorderRadius: 16
- Border: `color.withValues(alpha: 0.2)`
- BoxShadow: `color.withValues(alpha: 0.05)`, blurRadius 10, offset (0, 4)

**4 cards specification:**

| Card | Label | Icon | Color | Value format |
|------|-------|------|-------|-------------|
| OEE | "OEE" | `Icons.speed` | `_oeeColor(oee)` — see threshold table | `"${oee.toStringAsFixed(1)}%"` |
| MTBF | "MTBF" | `Icons.av_timer` | `_mtbfColor(mtbf)` — see threshold table | `"${mtbf.toStringAsFixed(1)}h"` |
| MTTR | "MTTR" | `Icons.build_circle_outlined` | `_mttrColor(mttr)` — see threshold table | `"${mttr.toStringAsFixed(1)}h"` |
| DISPONIB. | "DISPONIB." | `Icons.precision_manufacturing` | `_dispColor(disp)` — see threshold table | `"${disp.toStringAsFixed(1)}%"` |

**Color threshold helpers** (add to DashboardScreen, mirror KpisScreen pattern):
- `_oeeColor`: green if ≥ 85, orange if ≥ 65, red otherwise
- `_mtbfColor`: green if ≥ 48, orange if ≥ 24, red otherwise
- `_mttrColor`: green if ≤ 2, orange if ≤ 4, red otherwise (inverted — lower is better)
- `_dispColor`: green if ≥ 90, orange if ≥ 75, red otherwise

**Section label:** Insert section label `"KPIs OPERACIONALES"` using the existing `_buildSectionHeader` pattern (white text, 14px, bold, letterSpacing 1.1) above the two rows.

**Animation:** `.animate().scale(duration: 400.ms, curve: Curves.easeOut)` — same as existing `_buildKpiCard`.

**Data fetch:** Call `StatsService().fetchDashboardStats()` once on `initState` + manual refresh only. Do NOT add to the 5-second refresh timer. Store as `Map<String, dynamic>? _kpiStats`. Guard render with `if (_kpiStats != null)` — show nothing while loading (not an error state).

**Insertion point in DashboardScreen body:** After the existing `SizedBox(height: 8)` (correction: existing code has `SizedBox(height: 10)` — change to 8 to align with sm token) + `Row([_buildKpiCard("ESTADO PLANTA", ...), _buildKpiCard("ALERTAS CRÍTICAS", ...)])` and the `Row([_buildKpiCard("TOTAL TAREAS", ...), _buildKpiCard("OT PENDIENTES", ...)])`, before the `SizedBox(height: 30)`.

### Surface 2: fl_chart BarChart (KPI-03) — KpisScreen

**What:** Replace the existing `_buildEvolucionChart()` custom hand-rolled painter with a `fl_chart` `BarChart`. The wrapping Container and section header remain unchanged.

**Container wrapper:** Keep existing —
```
Container(
  padding: const EdgeInsets.fromLTRB(16, 16, 16, 12),
  decoration: BoxDecoration(
    color: IndustrialTheme.claudCloud,
    borderRadius: BorderRadius.circular(16),
    border: Border.all(color: Colors.white10),
  ),
  child: Column(children: [
    SizedBox(height: 160, child: BarChart(...)),
    SizedBox(height: 8),
    Row(/* legend — unchanged */),
  ]),
)
```

**BarChart dimensions:** `SizedBox(height: 160)` wrapping the `BarChart` widget.

**BarChart configuration:**
- `alignment`: `BarChartAlignment.spaceAround`
- `maxY`: computed as `max(preventivo, correctivo) * 1.25`, minimum 1.0
- Bar rod width: 8 logical pixels
- Bar rod border radius: `BorderRadius.circular(3)` (top corners only: `BorderRadius.vertical(top: Radius.circular(3))`)
- Rod colors: preventivo = `IndustrialTheme.operativeGreen`, correctivo = `IndustrialTheme.criticalRed`
- `BarChartGroupData.x`: index into `evolucionMensual` array (0–5)
- Each group has 2 `BarChartRodData` entries (preventivo first, correctivo second)

**Axes:**
- Bottom titles: `showTitles: true`, `reservedSize: 20`, `getTitlesWidget` returns `Text(evolucion[v.toInt()]['mes'], style: TextStyle(fontSize: 9, color: IndustrialTheme.slateGray, fontWeight: FontWeight.bold))`
- Left, top, right titles: `SideTitles(showTitles: false)`
- `gridData`: `FlGridData(show: false)`
- `borderData`: `FlBorderData(show: false)`

**Touch behavior:** `BarTouchData(enabled: false)` — no tooltips required for TFG demo.

**Empty guard:** If `evolucion.isEmpty`, render existing `SIN DATOS` container (height 160, centered white24 text) — do not pass empty list to `BarChart`.

**Animation:** `BarChart` animates entry by default in fl_chart. No additional `flutter_animate` wrap needed. Remove the `.animate().fadeIn()` wrapper that was on the old custom chart if it was added — fl_chart handles its own entry animation.

**Legend:** Reuse existing `_legend(color, label)` method unchanged.

### Surface 3: Sala de Servidores Widget (KPI-05) — DashboardScreen

**What:** New `SalaServidoresWidget` (self-contained `StatefulWidget` in `lib/widgets/sala_servidores_widget.dart`) embedded in DashboardScreen body.

**Insertion point in DashboardScreen:** Between the KPI mini-cards rows (Surface 1) and the existing `INCIDENCIAS ACTIVAS` section. Insert a `SizedBox(height: 20)` spacer before the widget, and add the section label `"SALA DE SERVIDORES"` using the existing header style above it.

**Visual structure:**

```
Container(
  padding: EdgeInsets.all(16),            // md
  decoration: BoxDecoration(
    color: IndustrialTheme.claudCloud,
    borderRadius: BorderRadius.circular(16),
    border: Border.all(color: Colors.white10),
  ),
  child: Column(crossAxisAlignment: CrossAxisAlignment.start, [
    Row([
      Icon(Icons.sensors, color: neonCyan, size: 18),
      SizedBox(width: 8),
      Text("EN VIVO", size: 9, w400, neonCyan, letterSpacing: 1),
      Spacer(),
      _LiveDot(live: _live),               // animated dot
    ]),
    SizedBox(height: 16),
    Row([
      _SensorCard(label: "TEMPERATURA", value: "${_temperatura?.toStringAsFixed(1) ?? '--'}°C", icon: Icons.thermostat, color: _tempColor(_temperatura)),
      SizedBox(width: 12),
      _SensorCard(label: "HUMEDAD", value: "${_humedad?.toStringAsFixed(1) ?? '--'}%", icon: Icons.water_drop_outlined, color: IndustrialTheme.neonCyan),
    ]),
  ]),
)
```

**`_SensorCard` sub-widget:** Same visual contract as `_buildKpiCard` — `Expanded`, `claudCloud` background, 16px radius, color-tinted border at `alpha: 0.3`, icon at top, value at 22px w700, label at 10px w700 white.

**Temperature color thresholds (`_tempColor`):**
- `operativeGreen` if temp < 25.0
- `warningOrange` if temp < 35.0
- `criticalRed` if temp ≥ 35.0
- `slateGray` if null

**Live dot (`_LiveDot`):**
- 8×8 `Container`, `shape: BoxShape.circle`
- Color: `operativeGreen` when `_live == true`, `criticalRed` when false
- When live: apply pulsing animation via `flutter_animate` — `.animate(onPlay: (c) => c.repeat()).scale(begin: Offset(1,1), end: Offset(1.4, 1.4), duration: 800.ms, curve: Curves.easeInOut).then().scale(begin: Offset(1.4,1.4), end: Offset(1,1), duration: 800.ms)`

**Loading state:** If `_temperatura == null && _humedad == null && !_live`: show inline `SizedBox(height: 60, child: Center(child: CircularProgressIndicator(color: neonCyan, strokeWidth: 2)))` in place of the two sensor cards.

**Poll interval:** `Timer.periodic(Duration(seconds: 5), ...)` — must call `_timer?.cancel()` in `dispose()`.

**API call:** `GET /api/plc/maquina/1` — no `?since=` parameter on first load. Take the last element of the returned list. If list is empty or HTTP error: set `_live = false`, do not clear existing values (show stale data with live dot turning red).

**Animation on entry:** `.animate().fadeIn(duration: 400.ms).slideY(begin: 0.05, end: 0)` on the outer container, same pattern as existing dashboard sections.

### Surface 4: "Exportar PDF" button (KPI-06) — KpisScreen AppBar

**What:** Add `ElevatedButton.icon` to `KpisScreen` AppBar `actions` list, to the LEFT of the existing `IconButton(Icons.refresh)`.

**Button specification:**
```dart
ElevatedButton.icon(
  onPressed: _stats == null ? null : () => _exportarPdf(),
  icon: const Icon(Icons.picture_as_pdf, size: 16),
  label: const Text('EXPORTAR PDF'),
  style: ElevatedButton.styleFrom(
    backgroundColor: IndustrialTheme.electricBlue,
    foregroundColor: IndustrialTheme.spaceCadet,
    padding: const EdgeInsets.symmetric(horizontal: 12, vertical: 8),
    textStyle: const TextStyle(fontSize: 10, fontWeight: FontWeight.bold, letterSpacing: 1),
    shape: RoundedRectangleBorder(borderRadius: BorderRadius.circular(8)),
  ),
)
```

**Disabled state:** When `_stats == null` (loading) — `onPressed: null` disables the button automatically. Flutter dims it with the theme's disabled color.

**Loading feedback during export:** Use `ScaffoldMessenger.of(context).showSnackBar(SnackBar(content: Text('Generando PDF...'), duration: Duration(seconds: 2)))` immediately on tap, before calling `PdfGenerator.generarKpiPdf(_stats!)`.

**Error feedback:** Wrap `_exportarPdf()` in try/catch. On error: `ScaffoldMessenger.of(context).showSnackBar(SnackBar(content: Text('Error al generar PDF. Comprueba los datos.'), backgroundColor: IndustrialTheme.criticalRed))`.

**Placement in AppBar actions:** `[ElevatedButton, SizedBox(width: 4), IconButton(refresh)]` — button before refresh.

**AppBar actions padding:** Wrap the button in `Padding(padding: EdgeInsets.symmetric(vertical: 8, horizontal: 4))` to prevent vertical overflow in AppBar.

## PDF Document Design Contract

These are the visual rules for the `pdf` (pw.*) package output. These are NOT Flutter widgets — they use the `pw.*` API.

### generarKpiPdf (KPI-06)

**Page format:** `PdfPageFormat.a4`, margin `pw.EdgeInsets.all(32)`

**Document structure (top to bottom):**

1. **Header block:**
   - `pw.Text('INFORME KPI — MÈLTIC GMAO', style: pw.TextStyle(fontSize: 18, fontWeight: pw.FontWeight.bold))`
   - `pw.Text('Generado: ${DateFormat('dd/MM/yyyy HH:mm').format(DateTime.now())}', style: pw.TextStyle(fontSize: 9, color: PdfColors.grey600))`
   - `pw.Divider(height: 16, color: PdfColors.grey400)`

2. **KPI values table:**
   - `pw.TableHelper.fromTextArray` with headers: `['KPI', 'VALOR', 'DESCRIPCIÓN']`
   - 4 rows: OEE / MTBF / MTTR / Disponibilidad with formatted values and Spanish descriptions
   - `headerStyle`: fontSize 9, bold
   - `cellStyle`: fontSize 9

3. **Distribución sección:**
   - Section label: `pw.Text('DISTRIBUCIÓN DE MANTENIMIENTO', style: pw.TextStyle(fontSize: 11, fontWeight: pw.FontWeight.bold))` with `pw.SizedBox(height: 8)`
   - `pw.TableHelper.fromTextArray` headers: `['TIPO', 'CANTIDAD', 'PORCENTAJE']` — 2 rows (preventivas, correctivas)

4. **OTs por estado sección:**
   - Same pattern — headers: `['ESTADO', 'CANTIDAD', 'PORCENTAJE']` — 3 rows (CERRADA, EN PROCESO, PENDIENTE)

5. **Evolución mensual sección:**
   - Section label + data table — headers: `['MES', 'PREVENTIVAS', 'CORRECTIVAS', 'TOTAL']` — up to 6 rows from `evolucionMensual`

6. **Ranking sección:**
   - Headers: `['#', 'MÁQUINA', 'INCIDENCIAS']` — up to 5 rows from `rankingIncidencias`

**Output file name:** `'KPI_Report_${DateFormat('yyyyMMdd_HHmm').format(DateTime.now())}.pdf'`

**Delivery:** `Printing.layoutPdf(onLayout: (format) async => bytes, name: fileName)` — works on Android (share sheet), Web (browser PDF viewer), Windows (print dialog). This is the same pattern used in `PdfGenerator.viewLocalPdf()`.

**Charts:** No charts in PDF. Data tables only. Per research: `pdf` package cannot render Flutter widgets and chart capture as bitmap is out of scope for TFG.

## Interaction States

| State | Visual | Where |
|-------|--------|-------|
| Loading (initial) | `CircularProgressIndicator(color: neonCyan)` centered | KpisScreen, DashboardScreen body |
| Loading (quiet refresh) | No spinner — data stays visible | DashboardScreen 5-second refresh |
| Loading (KPI stats only) | No spinner — KPI section simply absent (`if (_kpiStats != null)`) | DashboardScreen KPI row |
| Error | `Icons.wifi_off_rounded` (size 56, slateGray) + "ERROR DE CONEXIÓN" text (bold, white54, letterSpacing 2) + `ElevatedButton` REINTENTAR | KpisScreen full-screen error |
| Error (dashboard) | `Icons.wifi_off_rounded` (size 64) + "SERVIDOR NO DISPONIBLE" (14px, bold, white54) + error message (12px, white30) + REINTENTAR button | DashboardScreen full-screen error |
| Empty data | `SIN DATOS` text (Colors.white24) centered in 160px container | Chart/ranking sections |
| Sala de Servidores offline | `_live = false`, red dot, existing values shown stale or `--` if never loaded | SalaServidoresWidget |
| Export in progress | SnackBar "Generando PDF..." (2s) while `Printing.layoutPdf` runs | KpisScreen, OrdenesScreen |
| Export error | SnackBar with `criticalRed` background | KpisScreen, OrdenesScreen |
| Export disabled | `onPressed: null` (button auto-dims, no interaction) | When no data loaded |
